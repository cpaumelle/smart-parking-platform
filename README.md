## Overview

The Smart Parking Platform is a complete IoT solution for managing parking spaces using LoRaWAN sensors. It provides real-time occupancy tracking, data analytics, and device management capabilities.

### Key Features

- **ChirpStack Integration**: Built-in ChirpStack 4.14.1 LoRaWAN Network Server for complete control
- **Parking-Specific Services**: Priority-based display control, reservations, and space management
- **Real-time Data Processing**: Sub-200ms sensor-to-display actuation pipeline
- **Downlink Control**: Send commands and configuration to parking sensors
- **Bidirectional Control**: Automatic Class C downlink control for parking displays
- **SSL/TLS**: Automatic certificate management with Let's Encrypt
- **Production-Ready**: Docker-based microservices with PostgreSQL backend

### Technology Stack

| Layer | Technology |
|-------|-----------|
| **Frontend** | React, Vite, Nginx |
| **Backend** | Python 3.11, FastAPI, PostgreSQL 16 |
| **LoRaWAN** | ChirpStack 4.14.1, Mosquitto MQTT |
| **Infrastructure** | Docker, Traefik, PgBouncer, Redis |
| **Database** | PostgreSQL 16 with multi-schema design |
| **Proxy** | Traefik v3.1 with automatic SSL |

## Quick Start

### Prerequisites

- Ubuntu 24.04 LTS (or compatible Linux distribution)
- Docker Engine 24.0+ with Docker Compose v2
- Domain name with DNS access (e.g., `verdegris.eu`)
- Public IP address with ports 80, 443, 1700/udp open

### Installation

1. **Clone the repository** (or navigate to installation directory):
   ```bash
   cd /opt/smart-parking
   ```

2. **Configure environment variables**:
   ```bash
   sudo nano .env
   ```

   Update the following variables:
   - `DOMAIN`: Your domain name
   - `TLS_EMAIL`: Email for Let's Encrypt certificates
   - `POSTGRES_PASSWORD`: Secure database password
   - `CHIRPSTACK_API_TOKEN`: Generate from ChirpStack UI

3. **Create Docker networks**:
   ```bash
   sudo docker network create parking-network
   sudo docker network create web
   ```

4. **Start all services**:
   ```bash
   sudo docker compose up -d
   ```

5. **Verify deployment**:
   ```bash
   sudo docker compose ps
   ```

6. **Access ChirpStack**:
   - URL: `https://chirpstack.verdegris.eu`
   - Default credentials: `admin` / `admin`

### First Steps

1. **Configure ChirpStack**:
   - Create a Tenant
   - Create an Application
   - Generate API Token (User Settings → API Keys)
   - Update `.env` with `CHIRPSTACK_API_TOKEN`

2. **Register a Gateway**:
   - Navigate to Gateways → Add Gateway
   - Configure gateway to point to your VPS IP:1700

3. **Add a Device**:
   - Create Device Profile
   - Register Device with DevEUI, AppKey, etc.
   - Activate device

4. **Test Data Flow**:
   ```bash
   # Watch ingest logs
   sudo docker compose logs -f ingest-service

   # Trigger device uplink
   # Watch for incoming data
   ```

- **Port**: 8000 (internal)
- **URL**: `https://downlink.verdegris.eu`

  - `DELETE /downlink/queue/{dev_eui}` - Flush queue
  - `GET /devices/{dev_eui}` - Device details
  - `GET /applications` - List applications
  - `GET /gateways` - List gateways
  - `GET /health` - Health check

#### Parking Display Service
- **Port**: 8000 (internal)
- **URL**: `https://parking.verdegris.eu`
- **Purpose**: Real-time parking space state management with priority-based actuation
- **Endpoints**:

  - `POST /v1/actuations/manual` - Manual override to force display state
  - `GET /v1/spaces` - List all parking spaces with current states
  - `GET /v1/spaces/sensor-list` - Sensor DevEUI list for Ingest service caching
  - `POST /v1/reservations` - Create time-based reservation
  - `GET /v1/actuations/status/{space_id}` - Get current space status
  - `GET /health` - Health check
- **Features**:
  - Priority-based state engine (Manual > Maintenance > Reservation > Sensor)
  - Sub-200ms response time for sensor uplinks
  - Automated downlink control for Class C LoRaWAN displays

## API Documentation

### Downlink Service API

#### Send Downlink

**Endpoint**: `POST /downlink/send`

**Request**:
```json
{
  "dev_eui": "58a0cb00001019bc",
  "fport": 204,
  "data": "0003840096030540000500E02100",
  "confirmed": false
}
```

**Data Format**:
- **HEX**: Even-length string with characters `0-9a-fA-F` (e.g., `0102030405`)
- **Base64**: Standard base64 encoding (e.g., `AQIDBAUE`)

The service automatically detects the format and decodes accordingly.

**Response**:
```json
{
  "status": "queued",
  "dev_eui": "58a0cb00001019bc",
  "f_cnt": null,
  "fport": 204,
  "confirmed": false
}
```

#### Get Downlink Queue

**Endpoint**: `GET /downlink/queue/{dev_eui}`

**Response**:
```json
{
  "dev_eui": "58a0cb00001019bc",
  "total_count": 2,
  "items": [
    {
      "f_cnt": 42,
      "fport": 204,
      "confirmed": false,
      "data": "0003840096030540000500e02100",
      "is_pending": true
    }
  ]
}
```

#### Flush Queue

**Endpoint**: `DELETE /downlink/queue/{dev_eui}`

**Response**:
```json
{
  "status": "flushed",
  "dev_eui": "58a0cb00001019bc"
}
```

## Development

### Local Development Setup

1. **Clone repository**:
   ```bash
   git clone <repository-url>
   cd smart-parking
   ```

2. **Set up Python environment**:
   ```bash
   cd services/ingest
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. **Run service locally**:
   ```bash
   cd app
   uvicorn main:app --reload --port 8000
   ```

### Adding a New Service

1. **Create service directory**:
   ```bash
   mkdir -p services/my-service/app
   cd services/my-service
   ```

2. **Create Dockerfile**:
   ```dockerfile
   FROM python:3.11-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY app/ .
   CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
   ```

3. **Add to docker-compose.yml**:
   ```yaml
   my-service:
     build: ./services/my-service
     container_name: parking-my-service
     restart: unless-stopped
     environment:
       DATABASE_URL: postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres-primary:5432/${POSTGRES_DB}
     volumes:
       - ./services/my-service/app:/app
     networks:
       - parking-network
       - web
     labels:
       - "traefik.enable=true"
       - "traefik.http.routers.my-service.rule=Host(\`my-service.${DOMAIN}\`)"
       - "traefik.http.routers.my-service.entrypoints=websecure"
       - "traefik.http.routers.my-service.tls.certresolver=letsencrypt"
       - "traefik.http.services.my-service.loadbalancer.server.port=8000"
   ```

4. **Deploy**:
   ```bash
   sudo docker compose up -d my-service
   ```

### Code Style

- **Python**: Follow PEP 8, use `black` formatter
- **API Design**: RESTful principles, FastAPI best practices
- **Documentation**: Inline comments for complex logic, API docs via FastAPI
- **Logging**: Use structured logging with INFO level by default

## Security

### Access Control

- **PostgreSQL**: Password authentication, internal network only
- **Traefik**: HTTPS only, automatic certificate rotation
- **ChirpStack**: API token authentication
- **Services**: Internal Docker network isolation

### Firewall Configuration

**UFW Rules**:
```bash
sudo ufw allow 80/tcp    # HTTP (redirects to HTTPS)
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 1700/udp  # LoRaWAN Gateway
sudo ufw enable
```

### Secrets Management

- Store sensitive values in `.env` file
- Set file permissions: `chmod 600 .env`
- Never commit `.env` to version control
- Rotate API tokens and passwords regularly

### SSL/TLS

- Automatic certificate generation via Let's Encrypt
- Certificates stored in Docker volume: `traefik_certs`
- HTTP automatically redirects to HTTPS
- Certificate renewal handled by Traefik

## Backup & Recovery

### Database Backup

**Manual backup**:
```bash
sudo docker compose exec postgres-primary pg_dump -U parking_user parking_platform > backups/parking_$(date +%Y%m%d_%H%M%S).sql
```

**Automated backup script**:
```bash
#!/bin/bash
BACKUP_DIR=/opt/smart-parking/backups
DATE=$(date +%Y%m%d_%H%M%S)
sudo docker compose exec -T postgres-primary pg_dump -U parking_user parking_platform > $BACKUP_DIR/parking_$DATE.sql
gzip $BACKUP_DIR/parking_$DATE.sql
# Keep only last 30 days
find $BACKUP_DIR -name "parking_*.sql.gz" -mtime +30 -delete
```

**Schedule backup** (crontab):
```bash
0 2 * * * /opt/smart-parking/scripts/backup.sh
```

### Restore Database

```bash
gunzip -c backups/parking_20251007_020000.sql.gz | sudo docker compose exec -T postgres-primary psql -U parking_user parking_platform
```

### Configuration Backup

**Backup configuration**:
```bash
tar -czf config_backup_$(date +%Y%m%d).tar.gz config/ .env docker-compose.yml
```
