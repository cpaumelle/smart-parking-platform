## Executive Summary

Smart Parking platform built on a single VPS architecture, integrating proven IoT platform services (sensemy-iot-platform v4.0.2) with new parking-specific functionality. All services orchestrated via Docker Compose with Traefik reverse proxy handling SSL termination and routing.

### Core Services (Deployed ✅)

## Database Schema

**Database:** `parking_platform`

### Schemas:
- **core** - System configuration, users, roles
- **devices** - Sensor registration, device metadata (from IoT platform)
- **ingest** - Raw sensor data staging and uplink processing
- **analytics** - Historical data, aggregations (from IoT platform)
- **parking_config** - Parking sensor and display device registry
- **parking_spaces** - Parking space definitions, locations, and reservations
- **parking_operations** - Actuation audit trail and operational logs

### Additional Databases:
- **chirpstack** - LoRaWAN network server data (separate DB)

## Security Considerations

### SSL/TLS
- Automatic certificate generation via Let's Encrypt
- Certificates stored in `traefik_certs` volume
- HTTP automatically redirects to HTTPS

### Database Security
- PostgreSQL password in .env (chmod 600)
- Connection pooling via PgBouncer
- Internal network isolation

### Network Security
- UFW firewall enabled
- Only ports 80, 443, 1700/udp exposed externally
- Internal services communicate via Docker networks

### OVH API Security
- Credentials stored in `/opt/smart-parking/scripts/dns/ovh.conf` (chmod 644)
- Scoped to verdegris.eu domain only
- Read/Write permissions on DNS records

## Monitoring & Troubleshooting

### Health Checks
```bash
# Check all container status
sudo docker compose ps

# Check database connectivity
sudo docker compose exec postgres-primary pg_isready

# Check ChirpStack logs
sudo docker compose logs chirpstack --tail 50

# Check Traefik routing
curl -I https://chirpstack.verdegris.eu
```

### Common Issues

#### Traefik Not Discovering Services
**Solution**: Restart Traefik after services start
```bash
sudo docker compose restart traefik
```

#### Database Connection Failures
**Solution**: Check if PostgreSQL is healthy
```bash
sudo docker compose exec postgres-primary pg_isready -U parking_user
```

#### SSL Certificate Issues
**Solution**: Check Traefik logs for ACME errors
```bash
sudo docker compose logs traefik | grep -i acme
```
